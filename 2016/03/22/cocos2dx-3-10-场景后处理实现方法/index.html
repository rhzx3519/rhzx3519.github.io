cocos2dx 3.10 场景后处理实现方法

发表于 2016-03-22   |   分类于 Game   |   | 阅读次数   |   阅读次数

实现一个场景变灰的效果，首先，场景中主要有两个layer：mainLayer和uiLayer。mainLayer是场景中需要进行后处理的部分，而uiLayer是不需要后处理的。例子是在cocos的新项目的基础上修改的，可以通过cocos new projName -p packageName -l cpp命令创建一个新的工程。HelloWorldScene的代码如下：

新添加的部分解释一下：

  1. HelloWorld类中的新添加的成员变量RenderTexture使我们实现后处理的关键，它的作用相当于OpenGL中的帧缓冲，能够访问渲染场景并将其保存成纹理存储起来，用于之后的后处理。

  2. HelloWorld类中的这些layer 用于分层渲染。

  3. PostProcessing 这个精灵类用于保存RenderTexture中的纹理，并在其中进行后处理，具体实现后面再说明。

我们将两个sprite放在mainlayer中，而将ui按钮放在uiLayer中，注意mainLayer之上还要有一个mainLayerParent Layer。如果我们直接将mainLayer放在scene下面，则在之后的后处理中，我们需要将mainLayer设置成不可见，这样mainLayer就不会渲染了，我们也就得不到mainLayer渲染之后的纹理了。而将mainLayerParent设置成不可见，mainLayer仍然会渲染，我们就可以得到纹理了。
